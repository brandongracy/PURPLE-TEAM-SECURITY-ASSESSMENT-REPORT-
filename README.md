# PURPLE-TEAM-SECURITY-ASSESSMENT-REPORT-
This project simulates a real-world Purple Team engagement in an AWS environment with Active Directory.

The objective was to analyze both offensive and defensive perspectives by simulating attacker techniques while validating detection mechanisms using Windows logs, Sysmon, and AWS CloudWatch.
🎯 Objectives
Simulate real attacker behavior in a controlled environment
Validate detection using CloudWatch and Windows telemetry
Identify misconfigurations and attack paths
Map techniques to MITRE ATT&CK framework

🧪 Techniques Simulated
Network reconnaissance
Service enumeration
Credential reuse attacks
Lateral movement via WinRM
Brute force attempts (SSH)
Web application enumeration

🛡️ Detection & Monitoring
Windows Event Logs
Sysmon telemetry
AWS CloudWatch alerts
Security dashboards for authentication and network activity

📊 Key Findings
Exposed FTP with anonymous access
Sensitive data exposure in web directories
Multiple services exposed on a single host
Limited centralized logging visibility

🧠 Skills Demonstrated
Active Directory security
AWS security monitoring (CloudWatch)
Detection engineering mindset
Purple Team methodology
Attack simulation & analysis
